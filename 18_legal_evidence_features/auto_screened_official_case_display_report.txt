1. 작업 목적
사람이 원문과 대조한 verified 사례를 가장 우선하고, verified 사례가 없을 때에는 엄격한 구조·문자열·출처 품질검사를 모두 통과한 auto_screened 사례만 보조적으로 표시하도록 검색·검증·UI 구조를 개선했다. 깨진 OCR 문장을 복원하거나 추측하지 않았고, 품질 기준을 통과하지 못한 사례는 공개하지 않았다.

2. 기존 verified 사례 수
작업 전후 모두 0건이다. 기존 verified 상태와 검수 이력을 변경하지 않았다.

3. auto_screened 후보 수
기존 검수 저장소의 unverified 사례 75건을 후보 입력으로 검사했다. 75건은 모두 광산 관련성 high 또는 medium이었지만, 이것만으로 공개 승인하지 않았다.

4. auto_screened 최종 승인 수
0건이다. 엄격한 필수 조건을 모두 충족한 사례가 없어 auto_screened 상태로 승인하지 않았다.

5. 품질 미달 제외 수
75건 전부 제외했다. 승인 사례가 없더라도 기준을 낮추거나 깨진 문장을 임의 보정하지 않았다.

6. 제외 사유별 수
각 사례에서 가장 먼저 확인된 차단 사유 기준으로 메타데이터 품질 점수 미달 45건, 원본 카드 이미지 연결 누락 27건, display 사고 개요의 비정상 숫자·기호 연속 3건이다. case_id 중복 0건, content_hash 중복 0건, 출처 페이지 누락 0건, 한 글자 업종 0건, display 사고 개요 40자 미만 0건, 450자 초과 0건이었다. 원본 이미지 누락은 전체 기준으로 63건이지만, 이 중 36건은 앞 단계의 메타데이터 품질 미달로 먼저 제외됐다.

7. 직접 관련 검색 방식
질문 유형별 직접 용어군을 사용한다. 컨베이어는 컨베이어·벨트·끼임·말림, 차량은 후진·신호수·덤프트럭·굴착기·지게차·차량 충돌, 낙반은 낙반·암석 붕락·갱도 붕괴, 전기는 감전·전기설비·누전, 가스는 산소결핍·유해가스·질식, 발파는 발파·불발공·화약을 직접 관련 기준으로 사용한다.

8. 유사 위험 검색 방식
직접 용어가 없을 때만 사고 발생 원리 중심 용어군을 사용한다. 예를 들어 컨베이어 질문은 회전체·정비 중 재가동·에너지 차단, 후진 질문은 이동식 장비·사각지대·깔림, 낙반 질문은 토사·굴착면 붕괴·매몰, 전기 질문은 전원 투입·에너지 차단·잠금·표지, 가스 질문은 밀폐공간·환기 불량·가스 중독, 발파 질문은 폭발·폭발물·점화원을 유사 위험으로 구분한다. 관계없는 위험 유형은 반환하지 않는다.

9. 검색 우선순위
직접 관련 verified → 직접 관련 auto_screened → 유사 위험 verified → 유사 위험 auto_screened 순이다. 최대 3건만 선택하고 동일 case_id는 한 번만 반환한다.

10. 깨진 문자열 차단 규칙
알려진 OCR 잡음 토큰, 무의미한 영문 조각, Latin-1 깨짐, 한글 자모·과도한 글자 분리, 동일 문자·토큰 반복, 서로 다른 복수 날짜, 복수 사고 제목, 비정상 숫자·기호 연속, HTML·JSON·Python 리스트 형태, 통계표·홍보 문구를 보수적으로 차단한다. LOTO, PPE, CCTV 같은 허용 약어는 무조건 삭제하지 않는다.

11. display 필드 사용 방식
공개 카드, Gemini·하이브리드 참고 문맥, 핵심 주의사항은 display_accident_summary, display_cause_summary, display_prevention_summary만 사용한다. 사고 개요는 공백 제외 40~450자를 요구한다. 원인과 예방 필드는 원문에 없을 수 있으므로 빈 값 자체는 허용하되, 값이 있으면 같은 문자열 안전검사를 적용한다.

12. raw OCR 공개 차단 방식
현장관리자 공개 카드에서는 raw OCR, full OCR, 진단 JSON, text_hash, content_hash, 내부 점수와 내부 변수명을 읽거나 표시하지 않는다. 관리자·개발자 검수 화면에서만 원본 이미지와 OCR 원문을 대조할 수 있다. auto_screened 공개 카드에는 원문 expander를 제공하지 않는다.

13. verified 배지
“원문 대조 검증 완료”와 “원본 PDF와 대조하여 내용 검증이 완료된 공식 사고사례입니다.” 문구를 유지한다.

14. auto_screened 배지
“자동 품질검사 통과” 배지를 표시한다. 사람의 원문 대조 검수는 아직 완료되지 않았음을 별도 문장으로 명확히 알린다.

15. direct·analogous 배지
direct는 “직접 관련 사례”, analogous는 “유사 위험 사례”로 표시한다. analogous에는 동일 작업·설비 사고가 아니라 사고 원리와 위험요인이 유사한 사례라는 안내를 추가한다.

40. 교수님 설명용 문구
공식 사고사례는 사람이 원문을 직접 대조한 검증 완료 사례를 가장 우선해 표시하고, 해당 사례가 없을 경우 문자 깨짐, 문장 혼합, 출처 정보와 OCR 품질검사를 통과한 사례를 자동 품질검사 통과 사례로 구분해 제공합니다. 동일 작업의 사례가 없으면 끼임, 충돌, 붕괴, 질식, 폭발과 같이 사고 발생 원리가 유사한 공식 사례까지 단계적으로 검색합니다. 자동 품질검사 사례와 사람 검증 사례는 배지로 구분하며, 깨진 OCR 원문은 공개 화면에 표시하지 않습니다. 현재 데이터는 엄격한 기준을 통과한 사례가 없어 빈 결과를 유지했으며, 이는 품질 기준을 낮추지 않은 안전한 결과입니다.
