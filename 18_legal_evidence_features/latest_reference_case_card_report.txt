사례 매칭 방식:
- load_latest_reference_cases(): latest_reference_cases.json을 로드하며 파일 없음 또는 JSON 오류 시 빈 리스트를 반환합니다.
- match_reference_cases(question, intent, max_cases=2): 질문 intent/category 일치, 질문-사례 keyword 겹침, source_type 우선순위를 점수화해 최대 2개를 반환합니다.
- 범위 밖 질문(OUT_OF_SCOPE_INTENT)은 항상 빈 리스트를 반환합니다.
- official/public_agency는 우선 점수를 받을 수 있지만, 카테고리 또는 키워드가 맞을 때만 점수화되도록 보정했습니다.

UI 표시 방식:
- 답변 하단의 추천 증빙자료 아래에 별도 '최근 유사사례 참고' 섹션을 추가했습니다.
- 카드에는 사례 제목, 유형, 사례일/게시일, 출처명, 신뢰도 라벨, 요약, 시사점, 권장 조치, 주의 문구를 표시합니다.
- 주의 문구: 공식 법령 판단 근거가 아니라 이해를 돕기 위한 참고 사례이며 실제 판단은 담당 안전관리자 및 관계기관 확인이 필요하다고 표시합니다.

Gemini/하이브리드 반영 내용:
- Gemini 프롬프트에 관련 사례를 [최신 유사사례 참고]로만 전달합니다.
- 사례는 공식 법령 근거가 아니며 RAG 근거 문서처럼 단정하지 말라는 지침을 추가했습니다.
- 하이브리드 모드는 안정형 핵심 조치 구조를 유지하고, 사례는 이유 설명 보완에만 사용하도록 지시했습니다.

공식 RAG 근거와 참고 사례 분리 여부:
- RAG 근거 문서 섹션은 Vector DB 검색 결과만 표시합니다.
- 최신 유사사례는 별도 JSON 기반 카드 섹션에만 표시합니다.
- 대화이력/Excel 저장용 데이터에는 reference_cases 필드를 별도로 추가해 추천 증빙자료와 섞지 않도록 했습니다.
