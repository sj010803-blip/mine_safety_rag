네이버 뉴스 API 사용 방식:
- URL: https://openapi.naver.com/v1/search/news.json
- query, display, sort=date 파라미터 사용
- X-Naver-Client-Id, X-Naver-Client-Secret 헤더 사용
- requests가 있으면 requests를 사용하고, 없으면 urllib.request로 fallback
- timeout은 4초로 설정
- 제목, 요약, 게시일, 링크만 사용하며 기사 전문 크롤링은 하지 않음
- @st.cache_data(ttl=1800)로 query 기준 30분 캐시 적용

실시간 검색 결과 표시 방식:
- 범위 내 질문에서만 시도
- ENABLE_LIVE_CASE_SEARCH=true, LIVE_CASE_SEARCH_PROVIDER=naver, NAVER_CLIENT_ID/NAVER_CLIENT_SECRET이 있을 때만 검색
- 결과가 있을 때만 '실시간 사례 검색 참고' 섹션 표시
- 카드에는 기사 제목, 게시일, 기사 보기 링크, 요약, 신뢰도 라벨을 표시
- 주의 문구: 네이버 뉴스 검색 기반 참고자료이며 공식 법령 판단 근거가 아니라고 표시

API 키 미설정 시 fallback 방식:
- 키가 없거나 ENABLE_LIVE_CASE_SEARCH=false이면 조용히 빈 리스트 반환
- API 실패, 네트워크 실패, JSON 파싱 실패도 빈 리스트 반환
- 앱은 계속 동작하며 기존 사례 기반 주의 포인트만 표시

Gemini/하이브리드 반영 내용:
- 프롬프트에 [실시간 사례 검색 참고]를 뉴스 검색 참고자료로만 전달
- 공식 판단 근거는 RAG 근거 문서와 chunk_id라고 명시
- 뉴스 제목과 요약에 없는 내용을 만들지 않도록 지시
- 뉴스만으로 법령 위반 여부나 처벌 여부를 단정하지 않도록 지시

RAG 근거와 뉴스 검색 결과 분리 여부:
- RAG 근거 문서 섹션은 Vector DB 검색 결과와 chunk_id만 표시
- 네이버 뉴스 결과는 별도 '실시간 사례 검색 참고' 섹션에만 표시
- 기존 '사례 기반 주의 포인트'도 별도 작은 참고 섹션으로 유지

테스트 질문 결과:
1. 장비가 후진할 때 신호수는 왜 필요해? -> 장비 후진/신호수/충돌 방지 / live_cases=0 / 검색 결과가 없어도 오류 없이 기존 사례 기반 주의 포인트만 표시
2. 컨베이어 근처에서 작업할 때 뭘 조심해야 해? -> 컨베이어/회전체/끼임 / live_cases=3 / 실시간 사례 검색 참고 표시 대상
3. 소화기는 어디에 둬야 해? -> 화재/소화/비상대응 / live_cases=3 / 실시간 사례 검색 참고 표시 대상
4. 환기팬이 멈추면 어떻게 해야 해? -> 환기설비/팬/통풍 이상 / live_cases=3 / 실시간 사례 검색 참고 표시 대상
5. 천반에 균열이 보이면 어떻게 해야 해? -> 낙반/붕락/지보 / live_cases=1 / 실시간 사례 검색 참고 표시 대상
6. 중대재해처벌법 대응 증빙자료는 뭐가 필요해? -> 문서/증빙자료 추천 / live_cases=3 / 실시간 사례 검색 참고 표시 대상
7. 오늘 점심 뭐 먹을까? -> 범위 밖 질문 / live_cases=0 / RAG/KRAS/증빙자료/실시간 사례 검색 미표시
