10. app.py를 import하지 않은 이유

app.py는 Streamlit 상단 실행 코드, 모델과 Vector DB 로더, 외부 서비스 연결 준비, 파일 입출력 UI를 포함한 단일 애플리케이션 파일이다. import만 해도 상단 코드가 실행되어 세션 상태, 캐시, 외부 연결 또는 파일 접근이 발생할 수 있다.

따라서 import app, from app import ..., runpy.run_path(), exec(), subprocess 또는 Streamlit 실행을 사용하지 않았다. app.py는 정적 텍스트와 AST 구조로만 분석했다.

20. 정적 회귀 테스트의 한계

- 함수와 데이터 흐름이 존재하는지는 확인하지만 실제 실행 결과가 올바른지는 보장하지 않는다.
- AST 구조가 유지되어도 런타임 타입 오류, 파일 경로 오류, 세션 상태 오류가 있을 수 있다.
- source와 chunk_id 처리 코드가 존재하는지는 확인하지만 실제 검색 결과의 정확도는 확인하지 않는다.
- 법령 안전 문구가 존재하는지는 확인하지만 모든 생성 답변이 항상 해당 원칙을 지키는지는 보장하지 않는다.
- 비밀정보 검사는 명백한 직접 출력과 같은 함수 안의 정적 전달을 중심으로 한다. 동적 객체, 복잡한 컨테이너, 런타임 monkey patching까지 완전하게 추적하지는 못한다.
- 함수 이름과 현재 구조를 기준선으로 사용하므로 향후 안전한 리팩터링에서도 테스트가 실패할 수 있다. 이 경우 기능 손상인지 의도된 구조 변경인지 사람이 확인해야 한다.
- 정적 테스트 통과는 법률 검토 또는 현장 안전 검증 완료를 뜻하지 않는다.

21. 실제 앱 동작 테스트와의 차이

정적 회귀 테스트는 “핵심 코드 구조가 남아 있는가”를 빠르고 안전하게 확인한다. 실제 앱 동작 테스트는 Streamlit 화면을 열고 입력, 버튼, 세션 상태, RAG 검색, 외부 API, 파일 저장, 오류 fallback이 사용자 관점에서 작동하는지 확인한다.

이번 테스트는 app.py를 실행하지 않으므로 실제 화면 렌더링, 모델 로딩, DB 연결, 네트워크 시간초과, 저장 권한 문제는 검증하지 않는다. 향후 실제 앱 동작 테스트는 별도 whitelist와 보호 데이터 접근 승인 아래 독립 작업으로 진행해야 한다.

이번 단계에서는 새로운 기능을 바로 추가하기 전에 기존 MineSafe AI의 공식 문서 RAG, 질문 범위 분류, KRAS 위험성평가, 증빙자료 추천, 사례 기반 주의 포인트, 네이버 뉴스 참고 기능이 유지되는지를 자동 점검할 수 있는 비파괴 회귀 테스트를 구축했습니다. 테스트는 app.py를 실행하거나 수정하지 않고 정적 코드 구조를 검사하며, 향후 기능 확장 과정에서 기존 기능이 실수로 제거되는 것을 조기에 발견하기 위한 안전장치입니다.

26. 향후 Git whitelist 대상

사용자가 직접 검토 후 커밋할 후보는 다음 두 파일뿐이다.

1) 18_legal_evidence_features/test_minesafe_non_destructive_regression.py

최종 결론

허용된 두 파일만 생성했고 기존 파일은 수정하지 않았다. app.py의 작업 전후 크기, 수정 시각, SHA-256이 모두 동일하다. 최종 비파괴 정적 회귀 테스트는 14개 통과, 0개 실패, 0개 경고로 완료되었다. 이 결과는 핵심 정적 구조가 유지된다는 뜻이며, 실제 Streamlit 화면·RAG 검색·외부 API의 런타임 동작까지 검증했다는 뜻은 아니다.
