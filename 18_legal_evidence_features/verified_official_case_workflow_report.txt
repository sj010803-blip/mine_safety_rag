- verified 결과가 없으면 다음 취지로 안내한다.
  현재 질문과 관련해 원문 대조 검증이 완료된 공식 사고사례가 없습니다.
- 공개 카드에서 깨진 OCR 원문과 내부 OCR 진단값을 제거했다.
- 최근 뉴스 참고와 핵심 주의사항 탭, 범위 밖 질문 조기 반환은 유지했다.

5. 관리자·개발자 검수 화면
- 메뉴명: 공식 사고사례 검수
- 상단 지표: 전체 후보, 미검증, 검증 완료, 사용 제외, 수동검토
- 왼쪽: 원본 카드 이미지, 문서명, 페이지, case_id, 관리자 전용 OCR 원문
- 오른쪽: 발생일, 업종, 사고 유형, 사고 개요, 원인, 예방사항, 검증 메모
- 기능: 수정 저장, 수동 검토, 사용 제외, 검증 완료
- 검증 완료에는 원본 일치, 날짜, 업종, 사고 유형, 단일 사고, OCR 잡음 없음의 6개 확인이 모두 필요하다.
- 원본 카드 이미지가 없거나 필수 필드가 없으면 verified 저장을 차단한다.

- 컨베이어 끼임, 차량 후진·충돌, 낙반·붕괴, 전기·감전 유형별 최대 3건을 선정했다.

- 이미지와 검수 데이터는 25_verified_case_review 아래에 저장하며 Git에서 제외했다.
- 이미지 생성은 페이지를 300 DPI로 렌더링하고 Tesseract TSV 단어 좌표를 이용해 카드 영역을 나누는 보수적 보조 방식이다.
- 새 OCR 결과는 자동으로 verified 처리하지 않는다.

9. 레이아웃 OCR 대안 조사
- 현재 Python: 3.14.6
- PaddleOCR 3.7.0의 공개 배포 분류자는 Python 3.13까지 표시되어 현재 Python 3.14 환경과 호환 위험이 있다.
- 공개 패키지 기준 PaddlePaddle CPU Windows wheel은 약 101.7MB이며, PaddleOCR 본체 외에 PaddleX 의존성과 모델 다운로드가 추가된다.
- PaddleOCR 문서의 CPU 실행 경로가 있어 GPU는 필수가 아니지만, 레이아웃 모델별 실제 다운로드 용량은 선택 모델에 따라 달라진다.
- 기존 환경 충돌과 다운로드 규모를 고려해 이번 작업에서는 PaddleOCR/PaddlePaddle을 설치하지 않았다.

15. 남은 한계
