app.py 안의 작은 WORKER_EASY_TERM_EXPLANATIONS 상수로 표현 방향을 관리한다. 격리조치, 에너지 차단, 재가동 방지, 산소결핍, 유해가스, 비산분진, 불발공, 지보 상태와 작업 재개 승인을 일상적인 말로 바로 풀어 설명하도록 했다. 질문 유형별 지침도 컨베이어·차량·가스·불발공·분진·낙반·전기·관리체계로 구분했다.

13. 하이브리드 모드와의 차이
근로자 쉬운 설명 모드는 신규 근로자 대상의 짧은 행동·위험 이유·가상 예시 구조다. 하이브리드 모드는 현장관리자와 안전관리자 대상으로 안정형 초안을 유지하며 점검, 통제, 보고, 기록관리와 법적·관리적 맥락을 더 전문적이고 상세하게 설명한다. 두 프롬프트의 출력 제목과 필수 구조가 서로 다르며, 신규 테스트가 이 차이를 정적으로 확인한다.

14. 안정형 모드와의 차이
안정형 모드의 로컬 체크리스트 생성 구조는 변경하지 않았다. 안정형은 외부 Gemini 성공 여부와 관계없이 일정한 공식 문서 기반 핵심조치를 제공한다. 근로자 쉬운 설명 모드는 Gemini를 이용해 같은 공식 근거를 쉬운 교육 문장으로 재구성하며, 실패하면 안정형으로 돌아간다.

15. RAG 근거 충분성 연동
run_rag_flow에서 한 번 계산한 evidence_assessment를 근로자 쉬운 설명 프롬프트와 최종 출력 안전장치에 그대로 전달한다. 별도 재계산은 없다. 근거가 부족하면 정확한 수치나 시간을 말할 수 없다고 쉬운 문장으로 설명하고, 작업중지·접근금지·대피·안전관리자 확인을 안내한다. 기존 상태 배지와 관리자 진단, RAG 근거 영역은 유지했다.

16. Gemini 실패 fallback
기존 안정형 fallback을 유지했다. 근로자 쉬운 설명 생성에 실패하면 "쉬운 설명을 생성하지 못해 공식 문서 기반 핵심조치로 대신 안내합니다."라는 짧은 안내를 표시한다. 일반 사용자에게 내부 예외 전체나 인증정보를 표시하는 새 코드는 추가하지 않았다.

17. 범위 밖 질문 처리
기존 OUT_OF_SCOPE_INTENT 조기 반환을 유지했다. 범위 밖 질문은 RAG 검색, evidence 판정, Gemini 쉬운 설명 생성, 사례와 뉴스 조회보다 먼저 반환된다. 따라서 "오늘 점심 뭐 먹을까?"와 같은 질문에는 범위 안내만 표시된다.

18. 기존 대화 이력 호환성
기존 내부 GEMINI_MODE 값은 변경하지 않았다. normalize_answer_mode_label 함수가 과거 "자연어 설명 모드", "Gemini 모드", "natural", "gemini" 값을 표시 시 "근로자 쉬운 설명 모드"로 변환한다. 기존 JSONL 기록을 다시 쓰거나 삭제하지 않았다. 새 이력에는 answer_mode를 저장하고, 화면과 Excel에는 사람이 읽기 쉬운 "답변 모드" 열을 제공한다. 과거 기록에 값이 없어도 오류 없이 빈 값 또는 기록 없음으로 처리한다.

24. 실제 Gemini API 확인 여부
실제 Gemini 호출은 수행하지 않았다. 인증정보 값을 읽거나 출력하지 않았고, 외부 호출을 억지로 만들지 않았다. 프롬프트 분기와 fallback은 AST·정규식 기반 정적 테스트로 확인했다.

25. 확인하지 못한 사항
보호 규칙에 따라 실제 Vector DB와 chunk 내용을 열지 않았고 실제 RAG, 뉴스, Gemini 응답도 실행하지 않았다. 컨베이어, 메탄가스, 불발공, 접근 시간 질문과 범위 밖 점심 질문은 질문 의도별 쉬운 지침, 수치 생성 금지, 근거 부족 안전장치와 조기 반환의 정적 코드 경로로 확인했다. 실제 화면의 생성 문장 길이와 가독성은 실제 Gemini 응답 없이는 확인하지 못했다.

이 보고서는 해당 커밋에 함께 포함되는 파일이므로 작성 시점에는 커밋과 push 전이다. 최종 테스트, 기존 변경 지문과 staged whitelist가 모두 안전할 때만 커밋하고, main에 이번 작업 외 미푸시 커밋이 없을 때만 push한다. 실제 커밋 SHA와 push 결과는 작업 완료 보고에 기록한다. 동일 커밋의 SHA를 그 커밋 내부 파일에 미리 기록하는 것은 SHA가 파일 내용으로 결정되기 때문에 불가능하다.

34. 교수님 설명용 문구
기존 자연어 모드는 하이브리드 모드와 역할이 유사해 사용자가 차이를 구분하기 어려웠습니다. 이를 개선하기 위해 자연어 모드를 ‘근로자 쉬운 설명 모드’로 재설계했습니다. 이 모드는 신규 근로자와 비전문 작업자를 대상으로 전문용어를 줄이고, 지금 해야 할 행동과 위험한 이유를 짧은 문장과 쉬운 예시로 설명합니다. 반면 하이브리드 모드는 현장관리자와 안전관리자를 대상으로 체크리스트, 조치 이유, 관리상 주의사항을 더 상세하게 제공합니다.
