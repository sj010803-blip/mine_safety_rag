5. 위험 분야·세부 신호·작업 단계 추출 방식
- 정확한 질문 전체 문장을 비교하지 않고 분야별 기술 용어, 작업 대상, 상태 변화, 작업 단계 표현의 가중 조합을 사용한다.
- “발파”만으로 불발을 확정하지 않고 발파+작업 전+우천·발파공 물과 발파 후+불발·잔류약을 구분한다.
- “물”만으로 천반을 확정하지 않고 천반·천장·지보·출수 표현과 증가·변형 신호가 함께 있을 때 천반 출수 맥락으로 판단한다.
- “가스”만으로 메탄을 확정하지 않고 환기 정지, 정전, 측정값 상승, 산소 부족, 명시된 메탄을 별도 신호로 유지한다.
- “전기”만으로 LOTO를 확정하지 않고 정비 전, 전원 차단, 잠금·표지, 무전압 확인 조합을 사용한다.
- 작업 전, 시작 직전, 작업 중, 이상징후 발견 직후, 작업 후, 사고 발생 직후, 점검·정비 전, 작업 재개 전, 교육·회의, 불명확을 구분한다.
- 내부 정규화는 반복 공백과 “대처방안 수입→수립”, “광산안전전관리자→광산안전관리자”만 제한적으로 적용한다. 사용자 원문은 original_query에 그대로 보존한다.

- 기존 Vector distance는 원시값 그대로 유지한다. 퍼센트나 정확도 점수로 변환하지 않는다.

10. 공식 근거·수치·법령 방지
- 근거 요청 시 실제 evidence result의 source와 chunk_id가 함께 있는지 검사한다.
- 검색되지 않은 법령 조항 번호를 답변에서 발견하면 경고한다.
- 질문 또는 검색 근거에 없는 %, ppm, 시간, 거리, 횟수, 금액·형량 단위의 중요 수치를 발견하면 보완 또는 fallback 대상으로 처리한다. 목록 번호는 검사 대상이 아니다.
- 법령 위반·처벌을 확정하는 문장은 생성 계약에서 금지한다.
- 근거가 없으면 “공식 근거 보완 필요”와 담당 안전관리자·관계기관·전문가 확인을 안내한다.
