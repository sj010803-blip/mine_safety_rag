5. 질문 세부 위험 신호 추출 방식
- 원래 질문은 화면·이력용으로 유지하고 검색·분류용 normalized_question을 별도로 만들었다.
- 허용된 오타만 제한적으로 정규화했다.
  대처방안 수입 -> 대처방안 수립
  대처방안을 수입 -> 대처방안을 수립
  광산안전전관리자 -> 광산안전관리자
- 우천/강우 계열 + 발파 계열 + 발파 후/불발 표현 없음 조건을 rainy_pre_blasting으로 판정한다.
- 천반/지보 계열 + 유출수/출수/누수/지하수 유입 계열을 roof_water_increase로 판정한다.
- 발파 후/불발/불발공/잔류 화약 표현이 있으면 post_blasting_misfire로 유지한다.
- context에는 큰 위험 유형, 세부 위험 신호, 작업 단계, 요구 답변 형태, 표시 label을 분리해 저장한다.

6. 검색 질의 보강 방식
- Vector DB와 embedding은 변경하지 않았다.
- rainy_pre_blasting은 기존 불발공·발파 후 전용 확장어를 사용하지 않고 우천·강우, 낙뢰, 발파공 침수·공내수, 화약류·뇌관·점화설비 습윤·손상, 배수·접근로, 사면·갱구·천반·부석, 작업 연기·중지와 작업 전 점검을 보조 문맥으로 붙인다.

- 세부 신호가 있으면 내부 후보 수를 제한적으로 넓히고 해당 신호가 들어간 검색 결과를 우선 재정렬한다. 우천 발파 질문에서는 우천 관련 신호가 전혀 없는 불발 문서의 가중치를 낮춘다.
